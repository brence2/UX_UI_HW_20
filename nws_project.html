National Wearher Service Redesign


UI Design Case Study | September 2022

This Case Study was a project I did as part of the UX/UI Design Certification at UTSA.


MY ROLE

User Research, Definition & Ideation, Wireframing, Prototyping, Usability Testing

TOOLS

Figma, InVision, Miro, Google Docs, Adobe Color, WhatFont, ColorPicker

TIMEFRAME

4 Weeks

Problem and Solution


THE PROBLEM

The National Weather Service provides forecasts, safety, and general information for better protection of the lives and property of the citizens. However, a lot of people find the website to be confusing and cluttered with challenging navigation.



THE SOLUTION

Redesign the NWS website by highlighting the important content, making the site map less cluttered, and making the website responsive to devices of various sizes.

User Interface Analysis


User Path

The National Weather Service is an agency of the US federal government. It serves as a central hub for national weather information, providing access to organizations and the general public regarding weather forecasts, storm warnings, fire warnings, hurricane information, etc. Its goal is to provide up-to-date data for the purposes of protection, safety, and general information. Daniel visits Weather.gov in order to see weather forecasts and storm warnings that might affect flight routes. Daniel wants to avoid any discomfort for passengers and the aircraft team and make sure they fly safely.



Pages that the user visits:

Homepage: https://www.weather.gov/
Homepage with local weather: https://forecast.weather.gov/
Local weather details: https://forecast.weather.gov/
Forecast maps: https://www.weather.gov/forecastmaps
Wind speed and duration: https://graphical.weather.gov/sectors/conus.php?element=WindSpd

Proto-Persona


Data that the National Weather Service offers is highly important for Daniel. He browses the NWS website to get more advanced weather predictions. Based on those predictions, Daniel must make decisions and adjustments on flight routes to ensure flight safety.

Heuristics & Accessibility Evaluation


After reviewing several National Weather Service pages and checking heuristic evaluation data, my team and I concluded that most of the heuristics failed. On the other side, pages on the website load fast and we didn't find broken links.

Testing


After extracting NWS website colors using Adobe Color and performing Contrast Checker, all colors found on the website failed the contrast test.

In order to evaluate how effective the navigation of the NWS website is, I decided to conduct Guerrilla user testing. Tasks for users were:
Task 1: Check for the local weather forecast
Task 2: Check for severe weather forecasts
Task 3: Check for wind speed and directions
Task 4: Check for a Temperature map

Prioritization Matrix

Based on conducted user tests, users mostly had an issue with having too much data in one place. Some other issues were related to confusion (not knowing where to look for specific information on the website).


Redline & Annotations: Usability Issues

The navigation bar is cluttered with links
The interactive map legend contains too similar colors
Footer navigation is overwhelming with links all around the place


My initial idea when creating a Mood Board for the redesign of the NWS homage was to avoid using light-blue background and reduce the overuse of white space.


Information Architecture & UI Navigation Prototype


Annotated Navigation Analysis

Some of the most important steps that I decided to take after completing the annotation of the navigation were to :
Duplicate links need to be removed
The second nav bar needs to be removed
Footer needs to be reorganized
The map key needs to be alphabetized and moved to a better location.

Responsive Navigation Testing

The goal of the navigation testing was to find out the key pain points for users when navigating the current National Weather Service website on desktop and mobile devices. Users needed to complete several tasks:
Task 1: Navigate to the past weather page
Task 2: Navigate to the National Weather Service Enhanced Radar page
Task 3: Navigate to the Careers page
Task 4: Navigate to the Social Media News page (not just the social media links)

Some of the outcomes from that testing were:
Users were upset that they had to zoom in on their cell phones so they could find links
The home link from the main nav bar occasionally was not responding

Card Sorting

In order to evaluate the current grouping of the navigation bar for the National Weather Service homepage, I started Card Sorting and it was a real challenge. The first thing that I did was remove duplicate and quick links. After a few iterations, I grouped cards in the Main navigation bar with primary and secondary connections. After that, I also grouped footer links and utility links.

Site Map

To meet users' needs for logical and clear structure, and to better understand what navigation should contain, I created a Site Map for the NWS website. One of the major usability issues with the NWS website was the huge number of links and information organization. After performing card sorting, I simplified the Primary navigation by creating six primary categories.

Desktop Header Wireframes

As a navigation type for the NWS homepage, I used Dropdown Navigation Menu. I decided to go with this type of menu so users can see all links distributed evenly through the top of the screen without any difficulty.

Style Tile

In order to establish some rules and consistency for the redesign of the National Weather Service website, I created a simple Style Tile that I used throughout the project.

Responsive Style Guide

My goal in redesigning the NWS homepage was to add more colors and more content. The subscribe option was added to increase user interaction with the website. I also added a more engaging forecast card that registers users' location as soon as the user opens the website. Some other key aspects of the Stile Guide are listed below:

  • Gradients were used as the background and in the weather card components.
  • The NWS logo it's clearly visible properly on the light and dark background.
  • Typography, layout grids, input fields, and controls for the desktop and mobile versions of the homepage were adjusted so the content can fit properly.

Responsive Homepage Compositions Prototypes

The mobile version of the NWS homepage was redesigned to respond to the needs of the users and the devices they are using. Layout and images got accommodated to the screen size to improve visual experience and user interactions. The content was added in a single-column view.

The desktop redesign version of the NWS homepage also had a goal to respond to the needs of the users and the devices they are using. Layout and images got accommodated to the screen size to improve visual experience and user interactions. The content column range is a one to three-column view.

High-Fi Usability Testing

The primary goal of user testing on the redesigned NWS website was to determine if the redesign solves the original pain points and usability issues encountered during the previous testing. Understanding how users navigate through the complex navigation and what difficulties they may have with finding specific information will help in the iteration of the prototypes.

Iterated Mobile Homepage Composition

Based on the last testing, the following changes were made to the mobile and desktop prototypes:

  • Option to switch between C and F degrees added
  • Desktop primary navigation links changed to #1400FF color while hovered (previously were bolder)
  • Desktop primary navigation radius reduced to 5 degrees.

Final Thoughts / Conclusion

Working on this project was a great opportunity for me to learn the aspects of RWD. I wish I had more time to work on the sitemap. A good relationship between primary and secondary links helps users to find quickly what they need, and a well-designed sitemap is a key factor to make that happen.

The part of this project that I enjoyed the most is putting all pieces together and working on prototyping. Card sorting was the hardest step for me to do. I was struggling because the website has a lot of links on the homepage, and it was difficult to investigate which ones are duplicate links.

In the end, I think that I could implement a lot more features in my prototype. Because of the timeline for this project to be completed, I have a few more features on my mind to implement in this project
